# Инструкции для ИИ-ассистентов (AGENTS.md)

Данный документ содержит ключевые архитектурные правила, спецификацию структуры и стандарты разработки для приложения **Architector**. Все ИИ-модели, работающие с этой кодовой базой, должны строго соблюдать изложенные здесь соглашения.

---

## 🏛️ Архитектурная философия (Zero-Build)

1. **Отсутствие бандлера и сборки (Zero-Build):**
   * Приложение работает напрямую в браузере без Webpack, Vite, Rollup или Babel CLI.
   * Все глобальные библиотеки (React 18, ReactDOM, Babel Standalone, Tailwind CSS Play CDN, Lucide Icons) подключаются через CDN в `app/index.html`.
   * Компоненты подключаются тегами `<script type="text/babel" src="...">` в `app/index.html`.
   * **КРИТИЧЕСКОЕ ПРАВИЛО:** В файлах компонентов **НЕЛЬЗЯ использовать синтаксис ES-модулей `import` / `export`** для браузерного скриптинга. Все компоненты и утилиты объявляются в общем глобальном скоупе Babel.

2. **Двойной экспорт утилит (Browser + Node.js):**
   * Файлы `app/utils/geometry.js`, `app/utils/hierarchy.js` и `app/store/reducer.js` исполняются как в браузере, так и в средах без DOM (в тестовом раннере `node:test`).
   * В конце каждого такого файла используется схема двойного экспорта:
     ```js
     if (typeof module !== 'undefined' && module.exports) {
         module.exports = GeometryUtils;
     } else {
         window.GeometryUtils = GeometryUtils;
     }
     ```

## 📐 Модель данных v10 и Инварианты

1. **Формат сохранения (`formatVersion: 10`):**
   * Все координаты `position` у нод и слоев считаются **относительными** к верхнему левому углу родительского элемента (`parentId`).
   * Если `parentId === 'root'`, координаты совпадают с мировыми координатами холста.
   * Мировые координаты вычисляются только динамически вызовом `HierarchyUtils.getAbsolutePosition(id, nodes, layers)`.

2. **Ключи словарей:**
   * В объектах `nodes`, `layers`, `ports` ключ в словаре **ОБЯЗАТЕЛЬНО** совпадает с `entity.id` (например, `nodes["node-1"].id === "node-1"`).

3. **Форма нод (`shape`):**
   * Все узлы имеют стандартизированную прямоугольную форму `shape: "rectangle"`.

4. **Предотвращение двойного смещения:**
   * При перемещении родителя его потомки перемещаются автоматически за счет относительных координат. При массовом выделении проверяется `hasSelectedAncestor`, чтобы дочерние элементы не сдвигались дважды.

5. **Порядок скриптов в `index.html`:**
   * Скрипты подключаются strictly по цепочке: `utils/` → `store/` → `components/` → `app.js`. Новый компонент обязательно регистрируется в `app/index.html`.

## 🧪 Обязательное тестирование после изменений

После любых изменений в логике, редюсере, геометрии или компонентах **ИИ-ассистент обязан выполнить две команды проверки**:

1. **Юнит-тесты Node.js:**
   ```bash
   cmd /c "node --test app/tests/*.test.js"
   ```
   *Все тесты должны проходить со статусом PASS (0 ошибок).*

2. **Статическая проверка типов TypeScript:**
   ```bash
   cmd /c "npx -y -p typescript tsc -p app/jsconfig.json"
   ```
   *Команда должна завершаться с нулевым кодом ошибки.*

## 📌 Памятка для ИИ по стилю внесения правок

* **Не нарушать работу Zero-Build:** Не использовать `require()` или `import` в браузерных React-компонентах.
* **Использовать GitHub Flavored Markdown links:** При ответах пользователю создавать кликабельные ссылки на файлы вида `[basename](file:///absolute/path/to/file)`.
* **Сохранять docstrings и комментарии:** Не удалять существующие комментарии, описывающие бизнес-логику и математику выталкивания.
* **Использовать атомарные edits:** При правках файлов использовать точечную замену `replace_file_content` или `multi_replace_file_content`.
