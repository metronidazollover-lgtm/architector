# Architector — Visual Logic Node Editor

**Architector** — это интерактивный визуальный редактор иерархических графов (нодовых диаграмм), предназначенный для проектирования логики приложений, диаграмм потоков данных и архитектурных схем (например, цепочек промптов и интеграции ИИ-ассистентов).

## 🚀 Технологический стек
Проект выполнен по схеме "zero-build" и работает полностью в браузере без необходимости предварительной сборки:
* **Core:** React 18 & ReactDOM 18 (загружаются через CDN).
* **Компиляция JSX:** Babel Standalone (транспиляция кода на лету в браузере).
* **Стилизация:** Tailwind CSS (Play CDN с поддержкой кастомных стилей и директив `@layer`).
* **Иконки:** Lucide Icons (статический шрифт).
* **Сервер:** Локальный веб-сервер на Python (`server.py`, с заголовками no-cache — иначе браузер кэширует text/babel-скрипты и после правок исполняет смесь старых и новых файлов).

## 📂 Структура проекта

* `app/index.html` — главная точка входа. Настраивает базовые стили Tailwind, загружает библиотеки и подключает файлы компонентов в виде скриптов типа `type="text/babel"`. Новые компоненты добавляются сюда отдельной строкой script (порядок: utils → store → компоненты → app).
* `app/app.js` — корневой компонент приложения с предохранителем `ErrorBoundary` и рендером в DOM.
* `app/run.bat` — скрипт автозапуска. Запускает `server.py` на порту 3000 и открывает страницу приложения в браузере по умолчанию.
* `app/server.py` — сервер разработки с запретом кэширования.
* `app/store/reducer.js` — чистая логика состояния: defaultState, редьюсер, миграции формата, загрузка из localStorage. Без JSX, исполняется и в Node (тесты).
* `app/store/Store.js` — React-провайдер поверх редьюсера (`useReducer` + `Context API`). Поддерживает:
  * Сохранение состояния в `localStorage` (ключ `architector_state_v10`; legacy `architector_state_v9` конвертируется при первом запуске, см. `docs/MIGRATIONS.md`).
  * Историю действий (Undo/Redo на 20 шагов).
  * Операции с буфером обмена (Copy/Paste узлов).
  * Массовые операции (`MASS_UPDATE`, `DELETE_SELECTED`).
  * Навигацию по уровням иерархии: `DIVE_INTO` (вход в узел, порт или связь), `NAVIGATE_TO` (хлебные крошки), `GO_TO_CONTEXT` (прямой прыжок к контексту).
* `app/utils/hierarchy.js` — иерархия сущностей: `getAbsolutePosition` (мировые координаты из относительных), `getChildrenStats` (статистика вложенности), защита от циклов.
* `app/utils/geometry.js` — математический модуль для расчета геометрии графа:
  * `getPortAbsolutePosition` / `getPortRelativePosition` — расчет точных координат портов на гранях узлов.
  * `getSmartPlacement` — алгоритм автоматической раскладки узлов внутри слоя без перекрытий с динамической оптимизацией размера слоя под его контент (fit-to-content).
  * `resolveLayerCollision` — алгоритм разрешения коллизий (выталкивания) слоёв при перетаскивании, предотвращающий их наложение друг на друга.
* `app/components/` — React-компоненты интерфейса:
  * `Canvas.js` — холст диаграммы. Обрабатывает жесты трекпада, прокрутку колесика (зум в точку курсора), перетаскивание (Pan), клавиатурные сочетания (Ctrl+C, Ctrl+V, Delete и т.д.), хлебные крошки уровней и режим X-Ray (полноценное сквозное взаимодействие, перемещение и выделение объектов на просвечиваемых уровнях).
  * `Node.js` — визуальное представление узла. Поддерживает перетаскивание (включая активный контекстный узел вместе с содержимым), изменение размеров, привязку к сетке и двойной клик для входа внутрь (Dive In). Показывает количество детей иконкой `icon-boxes` в правом верхнем углу. Отображает реальное текстовое и медиа-содержимое при любом масштабе. Поддерживает умную автоподстройку размера под контент с возможностью ручного ресайза (флаг `userResized` защищает ручной размер от сброса при изменении названия или цвета).
  * `Layer.js` — контейнер-слой. Позволяет визуально группировать узлы и перемещать их совместно (замок в шапке слоя захватывает узлы, попадающие в его границы геометрически). Предотвращает перекрытие с соседними слоями на том же уровне иерархии за счет скольжения по границам при перетаскивании.
  * `Port.js` — точка подключения связей на границе узла.
  * `Link.js` — соединительная линия между портами (поддерживает кривые Безье и ортогональные линии).
  * `PropertyPanel.js` — панель редактирования свойств выделенных элементов, правый верхний угол (одиночное или массовое редактирование).
  * `Toolbar.js` — левая панель управления (добавление узлов/слоев, выравнивание слоев, экспорт/импорт JSON, изоляция выделенного, вызов AI-ассистента).
  * `Library.js` — панель-обозреватель проекта у левого края, рядом с тулбаром (список всех узлов, слоев и истории действий).
  * `AIAgentNodeContent.js` — интерфейс встроенного ИИ-ассистента для узлов типа `ai-agent` с возможностью автоматического изменения графа на основе JSON-команд.

* `app/types/` + `app/jsconfig.json` — JSDoc-типы; проверка: `npx -y -p typescript tsc -p app/jsconfig.json`.
* `app/tests/` — юнит-тесты чистой логики; запуск: `node --test app/tests/*.test.js`.

Планируемые изменения интерфейса и модели данных описаны в `docs/PLAN.md`; модель данных и инварианты — в `docs/ARCHITECTURE.md`; жесты и хоткеи — в `docs/UX.md`.

## 🛠️ Запуск проекта

1. Убедитесь, что у вас установлен Python (версии 3.x).
2. Запустите файл `run.bat` в папке `app` проекта.
3. Откроется окно браузера по адресу `http://localhost:3000`.
4. Для остановки сервера закройте свернутое консольное окно с именем `Architector Web Server`.

## 📝 Спецификация формата сохранений (JSON)

Проекты Architector экспортируются и импортируются в виде JSON-файлов со следующей структурой:

```json
{
  "formatVersion": 10,
  "layers": {
    "layer-id-1": {
      "id": "layer-id-1",
      "name": "Название слоя",
      "content": "Описание назначения слоя",
      "color": "#1d4ed8",
      "position": { "x": -2500, "y": -400 },
      "size": { "w": 900, "h": 950 },
      "locked": false,
      "parentId": "root",
      "snapToGrid": true
    }
  },
  "nodes": {
    "node-id-1": {
      "id": "node-id-1",
      "name": "Имя узла",
      "group": "Группа для классификации",
      "content": "Текст или логика внутри узла",
      "color": "#1e293b",
      "position": { "x": 70, "y": 100 },
      "size": { "w": 400, "h": 240 },
      "parentId": "layer-id-1",
      "snapToGrid": true,
      "shape": "rectangle",
      "type": "default"
    }
  },
  "ports": {
    "port-id-1": {
      "id": "port-id-1",
      "nodeId": "node-id-1",
      "type": "output",
      "edge": "right",
      "position": 0.5,
      "name": "Имя порта",
      "color": "#3b82f6"
    }
  },
  "links": [
    {
      "id": "link-id-1",
      "sourcePortId": "port-id-1",
      "targetPortId": "port-id-2",
      "name": "Название связи",
      "content": "Описание передаваемых данных",
      "color": "#3b82f6",
      "linkStyle": "bezier",
      "context": "root"
    }
  ],
  "canvas": {
    "offset": { "x": 100, "y": 200 },
    "zoom": 0.65
  },
  "currentContext": "root",
  "breadcrumbs": [
    { "id": "root", "name": "Главный холст" }
  ],
  "isolatedIds": []
}
```

### Детали полей:
* **Система координат (v10):** `position` относительна верхнему левому углу родителя; у элементов с `parentId: "root"` — мировые координаты холста. Ключ `id` внутри объекта всегда совпадает с ключом в словаре (при загрузке рассинхрон исправляется автоматически). Мировые координаты вычисляются подъёмом по цепочке `parentId` (`HierarchyUtils.getAbsolutePosition`).
* **Версия формата (`formatVersion`):** текущая — `10`. Файл без поля считается v9 (все позиции мировые) и конвертируется при импорте автоматически. Детали: `docs/MIGRATIONS.md`.
* **Камеры уровней (`cameraByContext`):** необязательный словарь `contextId -> { offset, zoom }` — сохранённый вид каждого посещённого уровня.
* **Иерархия (`parentId`):** Все слои и узлы имеют поле `parentId`. Если элемент лежит на главном экране, его `parentId` равен `"root"`. Если узел находится внутри слоя или другого узла, `parentId` равен ID этого родительского элемента. Циклы недопустимы; сироты с несуществующим родителем при загрузке относятся к `root`.
* **Форма узлов (`shape`):** По умолчанию `'rectangle'`. Все узлы имеют прямоугольную форму для высокой производительности и предсказуемости расчёта портов.
* **Грани портов (`edge`):** Указывает, на какой стороне узла расположен порт: `left`, `right`, `top`, `bottom`. Поле `position` задает смещение порта от начала грани (от `0.0` до `1.0`).
* **Типы узлов (`type`):** По умолчанию `'default'`. Тип `'ai-agent'` активирует интерактивную панель чата с ИИ-ассистентом.
* **Контекст связи (`context`):** справочное поле. Фактический уровень отображения связи редактор вычисляет из контекстов узлов на её концах; при расхождении вычисленное значение имеет приоритет. Внешним генераторам JSON достаточно корректно заполнить `sourcePortId`/`targetPortId`.
* **Изоляция (`isolatedIds`):** необязательный массив ID элементов, оставленных видимыми в режиме изоляции. Пустой массив или отсутствие поля — изоляция выключена.
