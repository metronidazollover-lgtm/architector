# Architector

Визуальный логический редактор узлов (Node Editor) в CAD-стилистике.

## Основной стек
- React 18 (без сборки, через CDN)
- TailwindCSS для стилизации
- Lucide Icons для иконок
- Пользовательская реализация Flux/Redux (React Context + useReducer) для управления стейтом

## Архитектура бота (Реализованные узлы)
- **Конфигурация**: Модули настройки новостей (News Config), трейдинга (Trade Config), отслеживания китов (Whale Tracker Config) и параметров Telegram-бота (TG Bot Config).
- **Источники сигналов**: AI News Analyzer, AI Strategy & TA, Whale Wallet DB, Top Coins DB Analyzer.
- **Ядро и риск-менеджмент**: Core Trading Engine, Risk Manager (контроль Position Sizing и Drawdown).
- **Исполнение и симуляция**: Exchange API (реальная биржа) и Paper Trading Engine (виртуальная среда/бектесты).
- **Мониторинг и UI**: Web Dashboard UI (веб-панель), Telegram Bot (управление с телефона), AI Assistant UI, Portfolio & State Tracker (синхронизация балансов), Alerts & Watchdog (модуль аварийной остановки/Kill Switch).

### Исправления уязвимостей архитектуры
1. **Paper Trading Engine**: Добавлена обратная связь в Portfolio для учета виртуального баланса и расчета PnL симуляции.
2. **Top Coins DB**: Настроено прямое обновление стаканов через вебсокеты от Exchange API.
3. **Безопасность (Watchdog)**: Внедрен Heartbeat-сигнал от Ядра. В случае его пропажи Watchdog имеет прямой доступ к Exchange API для экстренного закрытия позиций.
4. **Обучение ИИ (RL Feedback)**: Данные о фактических сделках из Portfolio отправляются обратно в AI Strategy & TA для адаптивного дообучения (Reinforcement Learning).
5. **Гонка состояний (Race Condition)**: Введено локальное резервирование маржи (Margin Lock ACK) между Ядром и Risk Manager для защиты от переторговки.

## Ключевые функции (MVP)
1. **Бесконечный холст**: Панорамирование (`Shift + ЛКМ` или СКМ) и зум (`Колесо мыши`).
2. **Узлы**: Добавление, выделение, перемещение.
3. **Порты**: Добавление, выделение, перемещение (Shift+Drag).
4. **Связи**: Добавление (Drag & Drop), выделение, удаление.
5. **Редактор свойств и размеров**: Изменение размеров узлов потягиванием за правый нижний угол, а также редактирование имени, цвета, текста и медиа в боковой панели.
6. **Библиотека (Обозреватель)**: Отображение всех построенных слоев, узлов, портов и связей на холсте.
7. **Слои (Фреймы)**: Возможность группировать узлы визуально через создание фоновых слоев с настраиваемым цветом и описанием (по аналогии с n8n / Miro).
7. **История (Undo/Redo)**: Отслеживание действий и возможность отмены/повтора.
8. **Импорт/Экспорт проекта**: Сохранение всей структуры (узлы, порты, связи) в JSON-файл и восстановление из него через Панель инструментов.
9. **Готовые шаблоны (Templates)**: Быстрая загрузка предустановленных архитектурных паттернов.
10. **Множественное выделение**: Выделение нескольких элементов с зажатым `Shift`. Групповое перемещение, удаление и массовое редактирование свойств (цвет, привязка к слою).
11. **Режим Изоляции**: Скрытие всех невыделенных элементов для фокусировки на определенной части архитектуры.
12. **Сквозное отображение (X-Ray / Deep View)**: Возможность включать видимость и интерактивность родительских или дочерних уровней напрямую из навигационных "хлебных крошек" (клик по иконке глаза). Это позволяет связывать порты узлов, находящихся на разных уровнях вложенности.
13. **AI Copilot (ИИ-Ассистент)**: Интегрированное диалоговое окно для работы с конфигурациями холста в виде полноценного универсального узла. 
   - **Окружение (Контекст):** Сбор контекста для анализа независим от режима работы. Вы можете выбрать:
     - *Глобально* — ИИ видит абсолютно все узлы проекта на холсте.
     - *Локально* — ИИ видит только те узлы, которые подключены к нему связями напрямую, а также все их внутренние вложенные элементы (рекурсивно).
   - **Режимы работы модели:**
     - *Agent (Агент)* — ИИ способен напрямую выполнять команды пользователя: создавать новые узлы, добавлять порты и связи, изменять свойства существующих узлов (текст, цвета, размеры) и размещать узлы внутри других узлов (меняя структуру графа через генерацию JSON-команд).
     - *Chat (Только чат)* — ИИ работает строго в режиме чтения (Read-Only). Он получает контекст холста, может консультировать и отвечать на вопросы по архитектуре проекта, но ему запрещено генерировать команды модификации графа.
   - Поддержка выбора моделей (GPT-4o, Claude 3.5, и др.), настройки API ключа и прикрепления медиа (изображений) к запросам.
