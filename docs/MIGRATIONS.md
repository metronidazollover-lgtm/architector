# Миграции формата сохранений

Формат описан в README (раздел «Спецификация формата сохранений»). Здесь — история версий
и правила конвертации. Код миграций живёт в `app/store/reducer.js` (`migrateToV10`),
тесты — в `app/tests/migration.test.js`.

## Версии

| Версия | Признак | Координаты |
|---|---|---|
| v9 | поле `formatVersion` отсутствует | все `position` мировые (абсолютные) |
| v10 | `formatVersion: 10` | `position` детей относительны верхнему левому углу родителя; у элементов с `parentId: "root"` — мировые |

## v9 → v10

Правило: `rel = absОld - parentAbsOld`, где позиции родителя берутся из исходных
(мировых) значений, поэтому порядок обхода не важен. Сирота с несуществующим `parentId`
остаётся с исходной позицией (трактуется как элемент корня).

Инвариант конвертации: мировая позиция каждой сущности до и после миграции совпадает
(проверяется тестом через `HierarchyUtils.getAbsolutePosition`).

Дополнительно при миграции:
- снапшоты undo/redo (`past`, `future`) сбрасываются: они содержат старые мировые координаты;
- `historyLogs` заменяется записью о конвертации.

## Точки входа миграции

1. **localStorage**: ключ `architector_state_v10`; при его отсутствии читается
   legacy-ключ `architector_state_v9` и конвертируется на лету. Старый ключ не удаляется.
2. **Импорт файла** (`LOAD_STATE`): файл без `formatVersion` считается v9 и конвертируется;
   файл с `formatVersion: 10` проходит без изменений.
3. **Экспорт** всегда пишет `formatVersion: 10`.

## Прочие изменения формата в v10

- Добавлено поле `cameraByContext` (камеры уровней), необязательное.
- Поле `links[].context` объявлено legacy: редактор его игнорирует и вычисляет контекст
  связи из портов; при экспорте поле сохраняется как есть только для обратной совместимости чтения старыми сборками.
