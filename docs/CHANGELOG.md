## 2026-07-24 — Рекурсивное погружение в порты, подсвечивание сетей связей и векторное вытягивание линий

- **Рекурсивное погружение в Порты (`DIVE_INTO`) и поддержка дочерних элементов:**
  - Реализовано погружение в порт по двойному клику (`state.currentContext = port.id`). Хлебные крошки строят путь: `Главный холст > [Имя Ноды] > Порт: <Имя>`.
  - Отрисовка на холсте (`Canvas.js`): при погружении в порт ярко подсвечиваются узел-владелец порта, соединенные связи (`opacity-100`, `zIndex: 10`), противоположные ноды/порты и дочерние элементы порта (`parentId === port.id`).
  - Создание дочерних структур: любые элементы, создаваемые внутри контекста порта, автоматически получают `parentId = port.id`.
  - Мировые координаты: `HierarchyUtils.getAbsolutePosition(id, nodes, layers, ports)` подтягивает координаты портов-родителей для дочерних сущностей.
  - Обозреватель проекта: вкладка «Дерево» (`OutlinerTree.js`) отображает порты под их узлами, а дочерние элементы — вложенными под строкой порта.
- **Визуальное подсвечивание связей, портов и узлов при клике:**
  - При выборе связи подсвечивается сама связь (`strokeWidth="6"` + drop shadow), а также соединенные порты (`scale-[3]`, white ring, glow) и ноды (`outline`, border, glow shadow).
  - При выборе порта подсвечивается порт, все присоединенные связи, противоположные порты и участвующие ноды.
  - Разделены понятия `isExplicitlySelected` (для обработчиков клика и стора) и `isSelected` (для визуальных стилей): клик по любому элементу записывает в `selectedIds` ровно 1 элемент, открывая для него одиночные свойства в `PropertyPanel` и мгновенно сбрасывая косвенную подсветку сети при переключении.
- **Исправление сдвига и динамическое векторное вытягивание связи (`PendingLink`):**
  - Устранен сдвиг координаты через `getBoundingClientRect()` контейнера `#canvas-container`.
  - Реализован векторно-направленный алгоритм кривой Безье в `Link.js`: временная связь вытягивается напрямую из центра порта `p1` строго к курсору мыши `p2` под любым углом движения (устранены петли на 100px вверх при движении вниз от верхних портов).
  - Заменено `transition-all duration-200` на точечные переходы `transition-[opacity,border-color,box-shadow,background-color]` в `Node.js`, `Port.js` и `Layer.js` для удержания мгновенного 0ms совпадения координат DOM с React-стейтом.
- **Тесты:**
  - Добавлены новые юнит-тесты в `reducer.test.js` для проверки погружения в порты и создания вложенных структур.
  - Все 58 тестов успешно проходят (`node --test`).

## 2026-07-19 — Автоподстройка размера нод, авторасстановка на слоях, коллизия слоёв

- **Автоподстройка размера ноды (calculateNodeSize):**
  - Калибровка высоты хедера: `40px` → `33px` (соответствует реальному CSS: `py-2`×2 + `text-sm` 14px + `border-b` 1px).
  - Введён флаг `userResized`: при ручном изменении размера ноды через resize-хэндл устанавливается `userResized: true`.
  - **Косметические правки** (имя, цвет, `snapToGrid`) не сбрасывают ручной размер — авто-size применяется как нижняя граница (`Math.max`).
  - **Контентные правки** (`content`, `mediaUrl`, `mediaHeight`) сбрасывают `userResized` и пересчитывают размер заново.
  - Авто-size применяется при `ADD_NODE`, `UPDATE_NODE` (поля name/content/mediaUrl/mediaHeight) и `LOAD_STATE`.
- **Привязка к сетке по умолчанию (snapToGrid):**
  - Все создаваемые или импортируемые ноды и слои теперь принудительно инициализируются с включенным свойством `snapToGrid: true` (применяется при ручном добавлении `ADD_NODE` / `ADD_LAYER` и импорте JSON `LOAD_STATE`). Это гарантирует ровное позиционирование элементов по сетке с самого начала. Пользователь по-прежнему может отключить его вручную в панели свойств для каждого элемента.
- **Авторасстановка нод на слое при назначении и загрузке:**
  - **При загрузке** (`LOAD_STATE`) все ноды внутри слоев по умолчанию выравниваются с помощью `getSmartPlacement` (например, при ИИ-построении или импорте неструктурированного JSON).
  - **Бесконфликтное размещение свободных нод:** Свободные ноды (которые не входят в слои, но находятся в одном контексте) автоматически расталкиваются от слоев на расстояние не менее **30px** и друг от друга на расстояние не менее **30px** с помощью алгоритма `GeometryUtils.resolveContextCollisions` (выполняется только при загрузке `LOAD_STATE`, чтобы не мешать ручному перетаскиванию).
  - **Автоматическое расталкивание слоев при загрузке:** Перекрывающиеся слои на одном уровне иерархии автоматически расталкиваются с зазором **30px** при `LOAD_STATE` с помощью алгоритма `GeometryUtils.resolveLayerCollisionsOnLoad` (позволяет избежать наложений при импорте хаотичных JSON).
  - **Кнопка и экшен «Выровнять слои» (ALIGN_LAYERS):** На левой панели Toolbar добавлена кнопка «Выровнять слои» (активна при наличии хотя бы 2 слоев в текущем контексте). При клике выстраивает все слои вертикально друг под другом с зазором **90px** по порядку номеров/имени (natural sort).
  - **Единичное назначение** (PropertyPanel → выбор слоя) теперь использует `getSmartPlacement` вместо фиксированной позиции `{x:40, y:90}` — ноды размещаются без перекрытий с существующими.
  - `getSmartPlacement` теперь **оптимизирует размер слоя** (fit-to-content): вычисляет bounding box всех дочерних нод + padding, вместо только расширения. Слой сжимается при удалении нод.
  - Увеличен вертикальный отступ сверху под шапкой слоя (`startY`): оставлен `90px` для компактного размещения, но теперь ноды автоматически выстраиваются ниже заголовка слоя.
- **Коллизия слоёв при перетаскивании:**
  - Новая функция `GeometryUtils.resolveLayerCollision`: при перетаскивании слоя проверяет перекрытие с соседями (слоями в том же контексте) и выталкивает по кратчайшему направлению с зазором 10px.
  - Итеративное разрешение (до 5 проходов) для каскадных случаев.
  - Слои из разных контекстов (разный `parentId`) не конфликтуют.
- **Тесты:**
  - Обновлены тесты auto-sizing под новую высоту хедера.
  - Добавлены тесты: `userResized` (2 теста), `getSmartPlacement` fit-to-content и no-overlap (3 теста), `resolveLayerCollision` (3 теста), `LOAD_STATE` автоматическое выравнивание на слоях (1 тест), `resolveContextCollisions` (2 теста), `GeometryUtils.alignLayers` + `ALIGN_LAYERS` (2 теста), `resolveLayerCollisionsOnLoad` (1 тест), `ADD_NODE` / `ADD_LAYER` / `LOAD_STATE` принудительный snapToGrid (3 теста).
  - Итого: 35 тестов, 0 провалов.
