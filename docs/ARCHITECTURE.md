# Архитектура Architector

Дополнение к README (структура файлов и запуск — там). Здесь — модель данных и инварианты.

## Слои кода

```
app/utils/geometry.js    чистая геометрия: порты, авторасстановка, коллизии
app/utils/hierarchy.js   иерархия: абсолютные координаты, статистика вложенности, циклы
app/store/reducer.js     defaultState, редьюсер, миграции, загрузка localStorage (без JSX)
app/store/Store.js       React-провайдер поверх редьюсера, персистенция
app/components/*.js      представление; читают стор через useStore()
```

Файлы без JSX имеют двойной экспорт (window + module.exports) и тестируются в Node:
`node --test app/tests/*.test.js`. Типы — JSDoc (`app/types/typedefs.js`), проверка:
`npx -y -p typescript tsc -p app/jsconfig.json`.

Скрипты подключаются в `app/index.html` по порядку utils → store → components → app.js.
Babel standalone исполняет их в общем глобальном лексическом скоупе, поэтому
`const`-декларации одного файла видны следующим.

## Модель состояния

Четыре словаря/списка сущностей: `layers`, `nodes`, `ports`, `links` (см. типы в
`types/typedefs.js`). Ключ словаря всегда равен `entity.id` (при загрузке рассинхрон
чинится автоматически).

### Иерархия

`parentId` образует дерево: `'root'` → слои и узлы → узлы внутри узлов/слоёв.
Контекст (`currentContext`) — узел, порт или связь, внутрь которых «нырнули»;
рендер фильтрует сущности по контексту (Canvas.js).

### Координаты (v10)

`position` относительна верхнему левому углу родителя; у детей root — мировая.
Следствия:

- перенос родителя перемещает всех потомков без обхода дерева;
- на экран всегда идут мировые координаты: `HierarchyUtils.getAbsolutePosition(id, nodes, layers)`
  (кэш на поколение стейта через WeakMap, защита от циклов parentId);
- смена родителя обязана пересчитывать позицию. Готовые пути: экшен `REPARENT_ENTITY`
  (сохраняет мировую позицию, отклоняет циклы) или явный расчёт через
  `HierarchyUtils.toRelativePosition`.

### Z-Index и Иерархия Рендеринга
Для корректного сквозного отображения вложенных структур (в т.ч. при включенном режиме X-Ray) Z-Index элементов на Canvas вычисляется на основе их глубины вложенности:
`nodeZIndex = (depth * 10) + (isPeek ? 30 : 0) + (isSelected ? 25 : 0) + ...`
Это гарантирует, что дочерние элементы всегда отрисовываются поверх родительских фреймов и слоев, предотвращая перекрытие кликов и графики.

### Пульсация родителей
При выделении дочернего узла его родитель переводится в режим пульсации (`isParentOfSelected`). Цветовая гамма пульсации динамически берется из цвета родительской ноды с помощью инлайновой CSS-переменной `--parent-pulse-color` и CSS-функции `color-mix`.

### Инварианты

1. `entity.id === ключ словаря`.
2. Цепочки `parentId` без циклов (обходы защищены `visited`-set, `REPARENT_ENTITY`
   отклоняет вложение в собственного потомка).
3. Мировая позиция сущности не меняется при: миграции формата, удалении родительского
   слоя (компенсация в `REMOVE_LAYER`/`DELETE_SELECTED`), перевложении через `REPARENT_ENTITY`.
4. `links[].context` не используется в логике: контекст связи вычисляется из контекстов
   узлов её портов (Canvas.js).
5. При выделении предка и потомка `MOVE_SELECTED` двигает только предка.

### Автоподстройка размера ноды

Функция `calculateNodeSize(name, content, mediaUrl, mediaHeight)` вычисляет оптимальный размер ноды:
- Header = 33px (py-2×2 + font-14px + border-1px + 1px запас).
- Content padding = 20px (10px сверху + 10px снизу).
- Средняя ширина символа = 8.5px, максимальная ширина = 794px (A4).
- Изображение: ширина ≥ 300px, высота += mediaHeight.

Флаг `userResized` на ноде:
- Устанавливается при ручном ресайзе через resize-хэндл.
- **Косметические правки** (name, color) — авто-size применяется как `Math.max` (нижняя граница), ручной размер сохраняется.
- **Контентные правки** (content, mediaUrl, mediaHeight) — `userResized` сбрасывается, размер пересчитывается заново.

### Авторасстановка на слоях

`getSmartPlacement` размещает ноды внутри слоя без перекрытий (padding = 20px, startY = 90px под шапку). После размещения **оптимизирует размер слоя** (fit-to-content): bounding box всех дочерних нод + padding. Минимумы: 300×200px.

При назначении ноды на слой через PropertyPanel вызывается `getSmartPlacement` как для единичного, так и для массового назначения.

Также автовыравнивание с помощью `getSmartPlacement` происходит **автоматически для всех слоев при загрузке проекта** (`LOAD_STATE`). Сразу после этого вызывается `resolveContextCollisions`, который выталкивает свободные ноды за пределы слоев на расстояние не менее **30px** и друг от друга на расстояние не менее **30px**. Это гарантирует, что ИИ-генерация архитектур и импортируемые неструктурированные JSON-файлы всегда будут загружаться выровненными, с красивыми отступами и без каких-либо перекрытий. При ручном редактировании это выталкивание не работает, чтобы не ограничивать пользователя в свободном размещении.

### Коллизия слоёв

`resolveLayerCollision` предотвращает перекрытие слоёв при перетаскивании:
- Проверяет перекрытие с siblings (слоями в том же контексте, `parentId`).
- Выталкивает по кратчайшему направлению с зазором (по умолчанию 10px).
- Итеративное разрешение (до 5 проходов) для каскадных случаев.

### История и навигация

- Undo/redo: снапшоты `{layers, nodes, ports, links}` в `past`/`future` (лимит 20).
- История посещений контекстов: `navHistory` (лимит 50), отдельно от undo;
  записи с удалёнными контекстами пропускаются при переходе.
- `cameraByContext`: камера каждого посещённого уровня, восстанавливается при возврате.

## Персистенция

`localStorage['architector_state_v10']` — весь стейт целиком после каждого изменения
(Store.js, с аварийным режимом при переполнении квоты). Экспорт/импорт файлов — Toolbar.js,
формат и миграции — docs/MIGRATIONS.md.
