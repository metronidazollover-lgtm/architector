# План: прозрачный UX для рекурсивных нод

Проект: Architector (`app/`), React 18 + Babel standalone, без сборки.
Все новые компоненты подключаются `<script type="text/babel">` в `index.html` (порядок важен: Store → utils → компоненты → app).

## Цель

Пользователь в любой момент понимает: где он в иерархии, что находится внутри каждого узла,
что связано с внешним миром, и как вернуться назад. Переходы между уровнями выглядят как
непрерывное движение камеры, а не телепорт.

## Текущее состояние (точка отсчёта)

Уже работает:
- `DIVE_INTO` по двойному клику (Node.js:79), хлебные крошки с уровнями (Canvas.js:313)
- X-Ray по номеру глубины (`TOGGLE_XRAY_LEVEL`), `GO_TO_CONTEXT` со сборкой пути
- Фильтрация рендера по `currentContext` (Canvas.js:473–546)
- Автоподстройка размеров нод под контент (высота хедера 33px) с флагом `userResized` (сохраняет ручной размер при косметических UPDATE_NODE и сбрасывает при контентных)
- Умная расстановка `getSmartPlacement` при единичном и массовом назначении ноды на слой с оптимизацией размеров слоя (fit-to-content)
- Коллизии слоёв при перетаскивании: скольжение стык-в-стык (resolveLayerCollision с зазором 10px) без перекрытий слоев в общем контексте
- Камера уровней (`cameraByContext`) и навигационная история (`navHistory`), undo/redo с прыжком в контекст правки
- Полная типизация JSDoc, 25 юнит-тестов (все зеленые) на `node --test`

Ключевые ограничения:
- Все координаты абсолютные, «вложенность» — фильтр по `parentId`. Перенос родителя не двигает детей.
- Узел с начинкой внешне неотличим от пустого.
- Камера при выходе сбрасывается в `{0,0, zoom:1}` (Store.js:339).
- Связь через границу уровня просто скрывается сверху (Canvas.js:439), контейнер её никак не отражает.

## Этап 0. Типизация и тесты — M

Выполняется после этапа 1 (чтобы не тормозить видимый результат), строго до этапа 2
(его миграция — главный потребитель и типов, и тестов).

### 0.1 Типизация через JSDoc, без отказа от zero-build
- TS-сборку не вводим. Типы живут в JSDoc-комментариях, проверка внешней командой:
  `npx tsc -p jsconfig.json` (`checkJs: true`, `noEmit: true`), код в браузере не меняется.
- `types/typedefs.js`: `@typedef` для NodeEntity, LayerEntity, PortEntity, LinkEntity,
  Camera, AppState, вариантов Action.
- `types/globals.d.ts`: объявления `window.GeometryUtils`, `window.HierarchyUtils` и глобальных
  компонентов (проект использует глобальные функции вместо модулей — tsc должен об этом знать).
- `strict: false` на старте, ужесточение по мере аннотирования. Начать с utils и Store.
- Мотив: этап 2 меняет семантику `position`; аннотации на `getAbsolutePosition` и всех читателях
  координат делают места преобразований явными и проверяемыми.

### 0.2 Юнит-тесты на встроенном node:test, без зависимостей
- Предпосылка: редьюсер выносится из store/Store.js в store/reducer.js без JSX
  (Store.js остаётся провайдером). В конце reducer.js двойной экспорт:
  `window.*` для браузера, `if (typeof module !== 'undefined') module.exports = ...` для Node.
- `tests/*.test.js`, запуск `node --test tests/`.
- Приоритет покрытия: экшены навигации (1.3–1.4), удаление с осиротением детей,
  `getClosestPointOnPolygon`, миграция v9→v10 (пишется вместе с этапом 2), защита от циклов
  reparent (этап 5).
- UI-тесты не заводим: ручной чек-лист приёмки остаётся, Playwright на этом размере не окупается.

## Этап 1. Быстрые победы (без смены модели данных) — S/M

### 1.1 Бейдж содержимого узла
- В шапке Node.js: счётчик прямых детей и внутренних связей, иконка «нырнуть».
- Селектор `getChildrenStats(nodeId)` в utils/geometry.js (или новый utils/hierarchy.js), мемоизация через `React.useMemo` от `state.nodes`/`state.links`.
- Приёмка: пустой узел без бейджа; узел с 3 детьми показывает «3»; клик по бейджу = dive.

### 1.2 Empty state контекста
- Если после dive в контексте нет ни узлов, ни слоёв: центрированная плашка
  «Вы внутри узла “X”. Добавьте элементы из библиотеки или нажмите Esc, чтобы подняться».
- Рендер в Canvas.js после списка узлов; условие — ноль детей у `currentContext`.

### 1.3 Клавиатурная навигация
- `Esc` = на уровень вверх (`NAVIGATE_TO` на предпоследнюю крошку).
- `Cmd/Ctrl+[` и `Cmd/Ctrl+]` = назад/вперёд по истории посещений контекстов.
- Store: `navHistory: { past: [], future: [] }`, пишется в `DIVE_INTO`/`NAVIGATE_TO`/`GO_TO_CONTEXT`;
  новые экшены `NAV_BACK`/`NAV_FORWARD`. История навигации отдельна от undo.
- Обработчики в существующий keydown-эффект Canvas.js:50 (там уже есть проверка на input/textarea).

### 1.4 Камера помнит каждый уровень
- Store: `cameraByContext: { [contextId]: { offset, zoom } }`.
- `DIVE_INTO` сохраняет текущую камеру для покидаемого контекста; `NAVIGATE_TO`/`NAV_BACK`
  восстанавливают сохранённую вместо сброса в ноль.
- `cameraByContext` попадает в экспорт JSON (спека в README уже фиксирует `canvas` и
  `breadcrumbs`, камеры уровней — их логичное продолжение); отсутствие поля при импорте — норма.
- Приёмка: нырнул, вернулся — вид ровно тот же, что до нырка.

## Этап 2. Фундамент: относительные координаты — L

Самое дорогое изменение. Делать до того, как схем станет много.

### 2.1 Модель
- `position` ребёнка хранится относительно верхнего левого угла родителя
  (для `parentId === 'root'` — как сейчас, мировые).
- Новый util `getAbsolutePosition(entity, state)`: подъём по цепочке `parentId` с защитой от циклов
  (паттерн `visited` уже используется в `getContextDepth`, Canvas.js:10).
- Кэш абсолютных позиций на кадр рендера (пересчёт при изменении nodes/layers).

### 2.2 Точки правки (все места чтения `position`)
- Canvas.js: рендер узлов/слоёв, `getTrueParentNodeId` не меняется.
- utils/geometry.js: `getPortAbsolutePosition` и авторасстановка.
- Store.js: `DIVE_INTO` (bbox), `CENTER_ON_ENTITY`, `MOVE_SELECTED` (двигаем только верхние
  из выделенных: если выделен родитель и ребёнок, ребёнок не двигается дважды).
- Layer.js: захват узлов при `locked` упрощается — дети слоя едут бесплатно.

### 2.3 Миграция
- `STORAGE_KEY` → `architector_state_v10`, конвертер v9→v10 в `getInitialState`
  (абсолютные → относительные одним проходом сверху вниз).
- `LOAD_STATE` (импорт файла): признак версии в JSON (`formatVersion: 10`); старые экспорты
  (`autobot1.json`, `kp_generator_project.json`) конвертируются на лету.
- Поле `links.context` объявляется legacy: редактор и сейчас пересчитывает контекст связи
  динамически (Canvas.js:427–446), в v10 поле перестаёт экспортироваться, при импорте игнорируется.
- Обновить раздел спецификации в README: система координат v10, `formatVersion`, `cameraByContext`,
  судьба `links.context`.
- Приёмка: оба существующих проекта открываются без визуальных смещений; перенос родителя
  двигает всю начинку; undo/redo работают на новых координатах.

### 2.4 Риск
- Производительность на больших графах: `getAbsolutePosition` O(глубина) на узел; кэш обязателен.
- Сломанные `parentId`-цепочки в старых данных: конвертер относит сирот к `root` (самолечение
  уже есть в getInitialState, расширить).

## Этап 3. Прозрачность: видеть внутрь — M

### 3.1 Превью внутренностей (semantic zoom)
- В теле узла (Node.js:157 уже переключает контент по `zoom < 0.4`) третий режим:
  при `zoom >= 0.6` и наличии детей — миниатюра начинки: детский bbox вписывается в тело узла
  (`transform: scale(k)`, `pointer-events: none`).
- Глубина превью: 1 уровень (узлы с именами, связи линиями). Рекурсивное превью
  превью не рендерим — заглушка-точки.
- Превью уважает `node.shape`: многоугольные узлы (triangle/pentagon/hexagon/octagon) рисуются
  через существующий `getPolygonPoints` (utils/geometry.js:2), не прямоугольниками.
- После этапа 2 миниатюра тривиальна: дети уже в системе координат родителя.
- Приёмка: на среднем зуме видно, что внутри узла; на дальнем — только имя и бейдж; fps не проседает
  (проверка на схеме kp_generator: ~30 узлов).

### 3.2 Peek по узлу (замена/дополнение X-Ray по уровням)
- `Alt+hover` над узлом с детьми: временная подсветка начинки этого конкретного узла в полный размер
  (opacity соседей вниз). Отпустил Alt — вернулось.
- X-Ray по глубине остаётся для power-режима, но из хлебных крошек его можно убрать позже.

## Этап 4. Миникарта рекурсии: «профильный срез» — M

Идея: видеть иерархию сбоку, как керн из скважины. Два инструмента в одной панели.

### 4.1 DepthProfile.js — вертикальный срез пути
- Панель в правом нижнем углу, сворачиваемая в узкую полоску-градусник глубины.
  Правый верхний угол занят PropertyPanel (PropertyPanel.js:65, `right-4 top-4 w-72`):
  профиль ограничен по высоте `max-h`, чтобы панели не перекрывались при выделении элемента.
- Одна строка = один уровень пути (root → ... → текущий контекст), сверху вниз.
- В строке — миниатюры всех «братьев» на этом уровне (дети контекста уровня):
  прямоугольники с усечёнными именами. Узел, через который проходит текущий путь, подсвечен
  акцентным цветом; текущий контекст — рамкой.
- Клик по любому брату на любом уровне = `GO_TO_CONTEXT` + `CENTER_ON_ENTITY`
  (оба экшена уже есть в Store.js). То есть смена ветки дерева в один клик без подъёма к корню.
- Hover = tooltip с именем и статистикой (из селектора 1.1).
- Братьев больше ~12 в строке: показываем первые 10 + чип «+N», клик по чипу раскрывает строку.
- Данные: путь берётся из `state.breadcrumbs`, братья — фильтр `parentId === crumb.id` по nodes.

### 4.2 MiniMap.js — пространственная карта текущего уровня
- Классическая миникарта под профилем: bbox всех сущностей текущего контекста, прямоугольник
  вьюпорта, drag по карте = панорама (`SET_CANVAS`).
- Узлы с детьми на миникарте помечены точкой — рекурсия видна и здесь.
- Рендер на `<canvas>` (не DOM) — дёшево перерисовывать при панораме.

### 4.3 Подключение
- Два новых файла в `components/`, две строки script в index.html, флаг `ui.depthProfileOpen`.
- Приёмка: на схеме autobot1 нырок на 2 уровня даёт три строки профиля; клик по брату
  на уровне 1 мгновенно перекидывает в его контекст с восстановлением камеры (этап 1.4).

## Этап 5. Структура: outliner, перевложение, прокси-порты — L

### 5.1 Вкладка «Структура» в Library.js
- Рекурсивное дерево всей иерархии: expand/collapse, имя + бейдж, клик = переход,
  двойной клик = центрирование.
- Drag-and-drop внутри дерева = reparent (см. 5.2).

### 5.2 Reparent
- Новый экшен `REPARENT_NODE { id, newParentId }`: пересчёт позиции в систему координат нового
  родителя (абсолютная позиция сохраняется, меняется только точка отсчёта), защита от цикла
  (нельзя вложить узел в собственного потомка).
- На холсте: перетащил узел на узел и задержал 400мс — подсветка «вложить», отпустил — reparent.
  Вытащил узел за границу родителя в режиме peek/X-Ray — поднялся на уровень выше.

### 5.3 Интерфейс контейнера — разбит на две части по риску

**5.3a Туннельные метки (делаем).** Внутри контекста по краям экрана — метки «→ наружу к узлу X»
для каждой связи, пересекающей границу; клик = переход к внешнему узлу. Декоративный
INPUT/OUTPUT-фон (Canvas.js:354) заменяется на эти рабочие метки. Только чтение: рендер связей
в Link.js не трогаем, правила видимости Canvas.js не трогаем. Риск низкий.

**5.3b Визуальные прокси-порты на границе (опционально, решение после этапов 1–4).**
Вычисляемые на лету порты-делегаты на грани контейнера, внешняя часть связи ведётся к ним.
В данные не пишутся, миграций не требуют; геометрия готова (`getClosestPointOnPolygon`).
Реальная сложность: разрезание связи на сегменты в Link.js поверх и так запутанных правил
видимости Canvas.js:404–468. Если после 5.3a потребность закрыта — не делаем;
запасной выход — бейдж «N внешних связей» на контейнере.

- Приёмка 5.3a: изнутри контекста видно и кликается каждое пересечение границы.
- Приёмка 5.3b (если делаем): связь изнутри узла к внешнему узлу видна на обоих уровнях;
  сверху — как связь контейнер→узел через прокси-порт.

## Этап 6. Гладкость — M

### 6.1 FLIP-переход dive/exit
- На время анимации (~400мс) смонтированы оба контекста: тело узла раздувается до вьюпорта,
  превью начинки (3.1) кросс-фейдится в реальный контент. Обратная анимация на выходе.
- Реализация: снапшот rect узла до, `requestAnimationFrame` + transform, класс-оверлей.

### 6.2 Zoom-to-dive
- Крутишь колесо внутрь узла с детьми: при пересечении порога (узел занял >85% вьюпорта и zoom
  растёт) — автоматический `DIVE_INTO`. Отдаление за порог — автоматический выход.
- Гистерезис порогов (вход 85%, выход 110%), чтобы не дребезжало на границе.
- Флаг в настройках UI: авто-нырок можно выключить.

### 6.3 Undo с прыжком в контекст
- Снапшот истории дополняется `contextId`; `UNDO`/`REDO` при откате правки из другого контекста
  сначала переходит туда (иначе изменение невидимо и undo выглядит сломанным).

## Сквозной сценарий приёмки (после этапов 1–4)

1. Открыть kp_generator_project.json — вид не отличается от текущего.
2. На среднем зуме видно превью начинки узлов, на дальнем — бейджи.
3. Нырнуть в узел колесом мыши (или двойным кликом) — плавный вход, крошки и профиль обновились.
4. В профиле кликнуть брата на верхнем уровне — мгновенный переход в его контекст.
5. Esc дважды — корень, камера в исходном виде (по 1.4).
6. Перетащить родительский слой — вся начинка едет вместе.
7. Ctrl+Z после правки внутри узла из корня — камера прыгает к месту правки.
