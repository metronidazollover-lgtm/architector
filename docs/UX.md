# Карта жестов и хоткеев

Обновляется каждым этапом плана (docs/PLAN.md).

## Мышь

| Жест | Действие |
|---|---|
| Клик по узлу/слою | Выделить узел/слой (одиночные свойства в PropertyPanel; визуально подсветить собственные порты, присоединенные связи, противоположные порты и соединенные ноды) |
| Клик по связи | Выделить связь + визуально подсветить соединенные порты (белое кольцо, масштабирование x3) и соединенные ноды (обводка и тень) |
| Клик по порту | Выделить порт + визуально подсветить присоединенные связи, противоположные порты и соединенные ноды |

| Shift+клик | Добавить/убрать из выделения |
| Перетаскивание узла | Переместить (потомки едут вместе, v10) |
| Перетаскивание шапки слоя | Переместить слой (замок: захват соседей по контексту в границах; слои автоматически скользят по границам друг друга, не допуская перекрытий) |
| Уголок справа внизу | Изменить размер |
| Двойной клик по узлу | Войти внутрь узла (Dive In) |
| Клик по бейджу «⧉ N» в шапке узла | Войти внутрь |
| Двойной клик по порту | Войти внутрь порта (Dive Into Port, просмотр и создание дочерних структур внутри порта) |
| Двойной клик по связи | Войти в контекст связи (Dive Into Link) |

| Средняя кнопка / Shift+ЛКМ по холсту | Панорама |
| Клик по пустому холсту | Сброс выделения |

## Автоподстройка и ручной размер узлов

- **Автоподстройка под контент:** При создании узла или изменении его текстового/медиа наполнения, его размер автоматически рассчитывается под объем текста и габариты картинки.
- **Сохранение ручного размера:** Если вы вручную изменили размер узла с помощью уголка справа внизу, этот размер фиксируется (устанавливается флаг `userResized`). При косметических изменениях (переименование, изменение цвета фона) этот ручной размер **не сбрасывается** (автоподстройка работает только как нижняя граница безопасности, чтобы текст не вылезал). Однако при значительном изменении наполнения (ввод нового описания, смена/удаление картинки) ручной размер сбрасывается, и узел снова подстраивается под контент.

## Авторасстановка на слоях и оптимизация размеров

- **Автоматическая раскладка:** При перетаскивании, импорте проекта/ИИ-генерации (в экшене `LOAD_STATE`), или назначении узлов в слой через правую панель свойств (как для одного узла, так и для нескольких), система автоматически вычисляет бесконфликтные координаты для нод, исключая их перекрытие.
- **Оптимизация (Fit-to-Content):** Слой автоматически подгоняет свои размеры (ширину и высоту) под занимаемую его элементами область плюс отступы (padding = 20px, верхний отступ под шапкой = 90px). Это работает как в сторону увеличения, так и в сторону уменьшения (слой аккуратно сжимается вокруг оставшихся в нём нод). Вы также можете запустить автовыравнивание в любой момент кнопкой «плитка» в шапке слоя.

## Предотвращение перекрытия слоёв (Коллизии)

- При ручном перетаскивании слоя за шапку, он взаимодействует с другими слоями на этом же уровне иерархии.
- Слои **не могут перекрывать друг друга**: если вы двигаете один слой на другой, перемещаемый слой начнет аккуратно скользить по границам соседнего слоя с зазором в 10px, выталкиваясь по кратчайшему пути.
- Это позволяет легко организовывать пространство и выстраивать слои стык-в-стык без хаотичного наложения.
- **Внешние отдельные ноды:** При автоматической генерации архитектур ИИ или загрузке проекта (`LOAD_STATE`) отдельные ноды, лежащие на одном уровне со слоями, автоматически расталкиваются от границ слоев на расстояние не менее **30px** и друг от друга на расстояние не менее **30px**. При ручном перетаскивании это ограничение не накладывается, чтобы пользователь мог свободно размещать элементы по своему желанию.
- **Предотвращение наложения слоев при импорте:** Перекрывающиеся слои автоматически расталкиваются с зазором **30px** при `LOAD_STATE`, чтобы ИИ-генерации не создавали наложенных друг на друга фреймов. Ваше собственное сохраненное ручное расположение (без перекрытий) сохраняется нетронутым.
- **Кнопка «Выровнять слои»:** При наличии 2 и более слоев в текущем контексте на левой панели тулбара появляется кнопка «Выровнять слои» (CAD-иконка вертикального выравнивания слоев). При нажатии слои выстраиваются по вертикали друг под другом с зазором **90px** по порядку номеров/имени (natural sort). Вы всегда можете отменить это выравнивание кнопками Undo/Redo или расположить слои вручную.

## Пульсация родительских узлов

- При выделении любого дочернего узла на любом уровне вложенности его родительский узел-контейнер начинает плавно и благородно пульсировать своим собственным цветом (`node.color`). 
- Пульсация происходит за счет анимации тени (`box-shadow`) и обводки (`outline`) с отступом 4px. Это помогает быстро понять, куда вложен выбранный элемент, не захламляя экран лишними рамками.

## Обозреватель проекта и Панель инструментов

- **Расположение панели Обозревателя:** Окно Обозревателя размещается у самого левого края экрана (`left-4 top-4`) с шириной **350px**. По умолчанию при старте чистый холст открывается с закрытым Обозревателем.
- **Динамический сдвиг Панели инструментов:** При открытии Обозревателя кнопкой на тулбаре Панель инструментов плавно сдвигается вправо на позицию `left-[382px]` (`transition-all duration-300`). При закрытии Обозревателя тулбар плавно возвращается в положение `left-4`.
- **3 Главные вкладки:** Шапка Обозревателя содержит 3 основных пункта: **«Объекты»**, **«Уровни»** и **«История»**.
  - **«Объекты»:** подвкладки *Дерево*, *Слои*, *Узлы*, *Порты*, *Связи*.
  - **«Уровни»:** иерархический список всех узлов-контейнеров с детальной статистикой вложенности (Уровень 0 — Главный холст, Уровень 1+) и прямым входом по двойному клику.
  - **«История»:** хронологический лог действий проекта с кнопками Undo/Redo.
- **Унифицированный клик закрытия:** Нажатие на кнопку «Обозреватель проекта» на Панели инструментов мгновенно сворачивает окно с одного клика независимо от активной вкладки.

## Миникарта (правый нижний угол)

- Отображает карту текущего уровня на canvas (контент, рамка вьюпорта).
- Клик или перетаскивание по миникарте осуществляет панорамирование холста.

## Навигация по уровням и X-Ray

- **Хлебные крошки сверху:** Клик — переход на уровень, иконка глаза — X-Ray просвечивание уровня. При переходе в пустой контекст выводится ненавязчивое сообщение `Пусто (Esc — наверх)`.
- **Взаимодействие на X-Ray уровнях:** Вы можете полноценно выделять, перемещать и редактировать любые элементы на просвечиваемых уровнях (Z-index автоматически выставляется по формуле `depth * 10`, исключая наложение родительских фреймов).
- **Перемещение контекстного узла:** Вы можете перетащить за шапку или тело активную ноду, внутри которой находитесь в данный момент — она сместится вместе со всеми внутренними детьми.
- **Сохранение камер:** Камера каждого уровня запоминается и восстанавливается при возврате.

## Тач

- Один палец по холсту — панорама, два пальца — зум (pinch).
